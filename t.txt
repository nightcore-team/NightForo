REST API endpoints (2.2)

GET alerts/
Gets the API user's list of alerts
Parameters:
- page (integer)
- cutoff (integer) - Unix timestamp of oldest alert to include
- unviewed (bool) - If true, gets only unviewed alerts
- unread (bool) - If true, gets only unread alerts
Response:
- alerts (UserAlert[])
- pagination (pagination)

POST alerts/
Sends an alert to the specified user. Only available to super user keys.
Parameters:
- to_user_id (integer) - ID of the user to receive the alert
- alert (string) - Text of the alert
- from_user_id (integer) - User to send the alert from
- link_url (string) - URL user will be taken to
- link_title (string) - Text of the link URL
Response:
- success (true)

POST alerts/mark-all
Marks all of the API user's alerts as read or viewed.
Parameters:
- read (bool) - Marks all alerts as read
- viewed (bool) - Marks all alerts as viewed
Response:
- success (true)

GET alerts/{id}/
Gets information about the specified alert
Parameters: None
Response:
- alert (UserAlert)

POST alerts/{id}/mark
Marks the alert as viewed/read/unread.
Parameters:
- read (bool) - Marks the alert as read
- unread (bool) - Marks the alert as unread
- viewed (bool) - Marks all alerts as viewed
Response:
- success (true)

GET attachments/
Gets the attachments associated with the provided API attachment key.
Parameters:
- key (string) - The API attachment key
Response:
- attachments (Attachment[])

POST attachments/
Uploads an attachment. Must be submitted using multipart/form-data encoding.
Parameters:
- key (string) - The API attachment key
- attachment (file) - The attachment file
Response:
- attachment (Attachment)

POST attachments/new-key
Creates a new attachment key.
Parameters:
- type (string) - Content type of the attachment
- context (string[]) - Key-value pairs representing context
- attachment (file) - First attachment to be associated
Response:
- key (string) - The attachment key created
- attachment (Attachment)

GET attachments/{id}/
Gets information about the specified attachment.
Parameters: None
Response:
- attachment (Attachment)

DELETE attachments/{id}/
Delete's the specified attachment.
Parameters: None
Response:
- success (true)

GET attachments/{id}/data
Gets the data that makes up the specified attachment.
Parameters: None
Response:
- data (binary) - The binary data

GET attachments/{id}/thumbnail
Gets the URL to the attachment's thumbnail.
Parameters: None
Response:
- url (string) - Via 301 redirect

POST auth/
Tests a login and password for validity. Only available to super user keys.
Parameters:
- login (string) - Username or email address
- password (string) - The password
- limit_ip (string) - IP that should be considered making the request
Response:
- user (User)

POST auth/from-session
Looks up the active XenForo user based on session ID or remember cookie.
Parameters:
- session_id (string) - Checks for active session
- remember_cookie (string) - Checks for active "remember me" cookie
Response:
- success (bool)
- user (User)

POST auth/login-token
Generates a token that can automatically log into a specific XenForo user.
Parameters:
- user_id (integer)
- limit_ip (string) - Locks token to specified IP
- return_url (string) - URL to return user after login
- force (bool) - Forcibly replace currently logged in user
- remember (bool) - Set "remember me" cookie
Response:
- login_token (string)
- login_url (string)
- expiry_date (integer)
